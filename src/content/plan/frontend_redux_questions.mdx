<details>
<summary>Что такое Redux и для чего он служит?</summary>
Redux - это библиотека, внешний стейт менеджер, для управления состоянием
в React(обычно). Служит для упрощения работы с состоянием в большом приложении.
</details>

<details>
<summary>Store в Redux</summary>
Store - это главный объект, в котором хранится глобальное состояние приложения.
</details>

<details>
<summary>Actions в Redux</summary>
Action - это объект, с обязательным полем "type". Action нужен для того,
чтобы дать сигнал редьюсерам каким-то определенным образом поменять
состояние, живущее в Store.
</details>

<details>
<summary>Reducers в Redux</summary>
Reducer - это функция, для изменения глобального сотояния. Более подробно, Reducer обрабатывает action, и возвращает новый state.
</details>

<details>
<summary>Для чего нужен Provider</summary>
Provider - это компонент, который обеспечивает доступ к Store.
</details>

<details>
<summary>Правила для редьюсеров в Redux</summary>
1. Не менять глобальное состояние напрямую - нужно создавать новое на основе старого состояния.
2. Возвращать новый state
3. Не вызывать асинхронные функции
4. Не вызывать функции, возвращающие рандомные значения
</details>

<details>
<summary>Какие хуки используются из библиотеки react-redux и для чего?</summary>
1. useSelector - позволяет получить состояние
2. useDispatch - позволяет изменить состояние
</details>
